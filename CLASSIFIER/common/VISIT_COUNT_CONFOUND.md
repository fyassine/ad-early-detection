# Visit-count confound in longitudinal early detection

Diagnostic notes for the `early_detection_table` AUC-vs-visits curves of the
longitudinal trajectory models (GEC, GELSTM, GEGRU). Generated by
`notebooks/SANITY/SANITY_VISIT_COUNT_CONFOUND.ipynb` (one run per model via the
`sanity-visit-confound-{gec,gelstm,gegru}` entries in `the experiments/ directory`), which
reloads each model's `latest/` trajectory run and calls the helpers in
`common/visit_confound.py` + `common/calibration.py`. The narrow-spread mitigations
(temperature-scaling calibration and a LayerNorm head variant) are **implemented** —
see "Spread mitigations" below.

## Findings

### Why N is not an apples-to-apples axis
`early_detection_table` ([common/early_detection.py](early_detection.py)) does **not**
hold the cohort fixed. For each `N` it keeps only subjects with `>= N` visits,
truncated to their first N, so the test cohort shrinks **34 → 28 → 18 → 12**. Each
row is a *different, self-selected, shrinking* population, so its AUC mixes three
effects: (a) genuine information per visit, (b) a shifting class balance, (c)
small-sample noise. The `cohort_composition_table` helper prints the per-N
converter/non-converter counts so this shift is visible beside the AUC.

### 1. GEC: the N=2 "leap" is real; the N=4 "drop" is a cohort artifact
On the **fixed** 12-subject cohort GEC rises **monotonically** (0.54 → 0.86 → 0.89 →
0.91): no N=4 drop. The variable-cohort "drop" (0.95 → 0.91) was the *changing
population*, not a 4th visit hurting. The N=2 leap survives the fixed cohort (0.54 →
0.86), so it is genuine Δt information — though in the variable table it is amplified
by single-scan subjects (near-chance) leaving the cohort at N≥2. **Do not rank models
on the variable N=4 row (n=12, one flip ≈ 0.08 AUC).**

### 2. GELSTM / GEGRU: AUC rises with N — confirmed on the fixed cohort
Both rise monotonically on the constant cohort (GELSTM 0.49 → 0.83 → 0.89 → 0.97;
GEGRU 0.63 → 0.80 → 0.89 → 0.94), so the recurrent gain from more visits is real, not
a cohort artifact. GELSTM ends highest at N=4 on these 12 subjects.

### 3. The structural confound — and its direction
Contrary to the initial hypothesis, in these runs **converters have *more* visits**,
not fewer: mean **3.43 vs 2.74** scans (Mann–Whitney U **p = 0.002**;
`summarize_visit_counts`). A subject is `converter` if *any* visit is "converter"
([DATA/src/splitting/create_downstream_data_splits.py](../../DATA/src/splitting/create_downstream_data_splits.py)),
and here converters are followed longer. So as N grows the early-detection cohort gets
*enriched* for converters (`frac_converter` 0.41 → 0.50 → 0.56 → 0.58, and N=5–6 are
converter-only — `cohort_composition_table`), which is exactly why the variable-cohort
per-N AUCs are not comparable.

**Is this a visit-count shortcut, or evidence accumulation?** Between subjects, the
*overall* prob~`n_scans` correlation is ~0 (GEC +0.17 n.s., GELSTM −0.11 n.s.,
GEGRU −0.08 n.s.) — the marginal trend (converters followed longer) and the
within-non-converter trend cancel (Simpson's paradox). The decisive test is the
**within-subject slope** (`within_subject_prob_slopes`): how P(converter) moves as a
*single subject's own* visits accumulate. A count shortcut would be flat
within-subject; instead every model shows **evidence accumulation** — converters drift
**up**, non-converters drift **down**:

| model | within-conv median slope (frac<0) | within-nonconv median slope (frac<0) |
|------:|----------------------------------:|-------------------------------------:|
| GEC   | +0.13 (0.00) | −0.05 (0.79) |
| GELSTM| +0.02 (0.21) | −0.06 (0.86) |
| GEGRU | +0.06 (0.14) | −0.20 (0.86) |

So ~80–86% of non-converters get *more confidently stable* as their own follow-up
grows — each extra stable visit is evidence, not a length cue. (The RNNs never receive
the count as a feature; GEC's `append_visit_mask` is the only explicit length signal.)
Corroborate with the `sanity-gelstm-ablations` shuffle-order / zero-Δt run.

### 4. Probability spread: narrow RNNs vs wide GEC, and how to widen it
Raw converter-vs-non-converter separation: **GEC 0.53**, **GEGRU 0.34**, **GELSTM
0.09** — GELSTM is by far the tightest. This is architectural, **not** a missing
`pos_weight` (all three use `BCEWithLogitsLoss(pos_weight=n_neg/n_pos)`). The real
causes: the RNN compresses the trajectory into a **32-d hidden state**, and its head is
shallow `32→32→1` with **no normalisation** — vs GEC's `[256,128,64]` head with
**BatchNorm after every layer** ([adapters/gec.py](../adapters/gec.py)), which produces
sharper logits. There was also **no calibration** anywhere. GELSTM ranks well (high
AUC) but logits stay small → sigmoid clusters near 0.5.

## Spread mitigations (implemented)

Two complementary fixes, both shipped:

**(a) Temperature scaling** (`common/calibration.py`) — a single scalar T fit on the
OOF predictions at training time (`LONGITUDINAL_COMMON_DELCODE.ipynb` writes
`calibration.json`), applied post-hoc. **Monotonic ⇒ AUC and thresholded decisions
unchanged**; it only widens the spread and improves ECE.

**(b) LayerNorm head variant** (`classifier_norm: "layernorm"`, wider `lstm_hidden`/
`classifier_hidden`, lower dropout — `configs/gelstm_whole_brain_wide_lnorm.json`,
experiment `gelstm-whole-brain-wide-lnorm`) — a retrained model that produces a
GEC-like wide spread natively.

Test-set results (`sanity-visit-confound-*`):

| model | separation raw → cal | T | test ECE raw → cal | test AUC |
|------:|---------------------:|--:|-------------------:|---------:|
| GEC        | 0.53 → 0.52 | 1.04 | 0.225 → 0.229 | 0.96 |
| GELSTM     | 0.09 → **0.14** | 0.64 | 0.254 → **0.174** | 0.81 |
| GEGRU      | 0.34 → 0.37 | 0.74 | 0.236 → 0.269 | 0.83 |
| GELSTM wide+LN | **0.50** → 0.46 | 1.43 | 0.200 → **0.164** | 0.82 |

Takeaways: temperature scaling lifts GELSTM's separation ~50% (0.09→0.14) and clearly
improves its calibration (ECE 0.254→0.174) for free; the **LayerNorm variant** is the
real structural fix — raw separation 0.50 (≈ GEC) and the best raw ECE, at comparable
test AUC (0.82 vs base 0.81). GEC is already confident (T≈1) so calibration barely
moves it. (All numbers regenerate via the four `sanity-visit-confound-*` runs.)

## Other mitigations (documented, not implemented)

1. **Report the fixed cohort + per-N class balance by default** wherever an AUC-vs-N
   table is shown (the longitudinal notebook + dashboard), via
   `early_detection_fixed_cohort` / `cohort_composition_table`.
2. **Confidence intervals on small-N rows** (bootstrap) and don't rank models on N=4
   (n=12) — would extend `common/early_detection.py`.
3. **Control for follow-up depth** — a follow-up-matched converter/non-converter
   subset, or visit count as an audited covariate (splitting/eval layer). Given the
   within-subject slopes show evidence accumulation rather than a count shortcut,
   dropping `append_visit_mask` is *not* indicated.
